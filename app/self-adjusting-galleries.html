Self Adjusting Galleries

When working with galleries, we run into problems when we can't control the exact number of gallery items. That's the biggest problem when working with galleries. Sometimes we end up with ugly grids like this :(

That sucks. One of the ways we can fix this is to make sure the last few items are spread out evenly to cover the full width. We can do so with some selector-jitsu mastery and some complex double loops (see code).

Gallery Reloaded to the rescue!

Gallery reloaded takes in a boolean in the second argument in addition to what we discussed in gallery reloaded. 

.gallery__item {
  @include gallery-reloaded(3, bottom gallery__item true);
}

Gallery Reloaded also manages to pacify other variations in the grid, as long as the total columns is divisible by the number of stray items.
